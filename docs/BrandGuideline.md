# BrandGuideline.md

## Brand Name

GeoChem Suite

## Brand Logo

### Primary Logo

Located in:
`/assets/branding/`

Use the official GeoChem Suite logo asset from this folder only.

### Logo Rules

- Do not distort or stretch the logo.
- Maintain original proportions.
- Use high contrast backgrounds only.
- Ensure clear visibility on light and dark surfaces.
- Minimum clear space: height of the “G” on all sides.
- Preferred placement: top-left navigation, login screens, report headers.

## Brand Positioning

GeoChem Suite is a premium laboratory operations platform for geochemical workflows and analytical labs.

Perception targets:

- Scientific precision
- Enterprise reliability
- Clean operational clarity
- Modern technical system

## Brand Personality

- Premium
- Clean
- Modern
- Confident
- Minimal
- Scientific
- Data-driven
- Precise

## Color System

### Primary

Cyan Blue
#06B6D4

### Secondary

Gold
#FBBF24

### Neutral System

- Background: transparent / clear base (#FFFFFF with controlled opacity layering)
- Surface: #F8FAFC
- Text Primary: #0F172A
- Text Secondary: #475569
- Borders: rgba(15, 23, 42, 0.08)

### Usage Rules

- Use opacity layering for depth instead of heavy gradients.
- Blend cyan and gold sparingly for highlights only.
- Maintain neutral dominance in layouts.
- Avoid saturated noise or competing colors.
- Background must remain visually clean and unobstructed.

### Semantic Colors

- Success: #16A34A
- Warning: #F59E0B
- Error: #DC2626
- Info Accent: Cyan Blue primary

## Typography

### Font Families (Dual System)

- Primary Font: Inter (UI, headings, core text)
- Secondary Font: IBM Plex Sans (data tables, system labels, technical sections)

### Heading Style

- Inter SemiBold / Bold
- Clear hierarchy with consistent scale

### Body Style

- Inter Regular

### Data / Technical Text

- IBM Plex Sans Regular / Medium

### Button Text

- Inter Medium

## Layout

- Reduced horizontal padding for tighter information density
- More compact page width for dashboards
- Maintain breathing space between modules, not within containers
- Align content closer to grid edges for efficiency
- Avoid excessive whitespace between sections

### Spacing Rules

- Base unit reduced by ~15–20% from standard enterprise layouts
- Prioritize data visibility over decorative spacing

## UI Style

- Premium enterprise scientific interface
- Clean, structured, data-first design
- Soft shadows only where needed for separation
- Rounded corners (controlled radius)
- Neutral-heavy surfaces with cyan/gold accents

Avoid:

- Neon styling
- Heavy gradients
- Decorative visual noise
- Gaming-style UI elements

## Components

### Buttons

#### Primary

- Cyan Blue background
- White text
- Subtle hover darkening

#### Secondary

- Transparent or neutral background
- Cyan border accent

#### Hover

- Slight elevation
- Soft color transition

#### Disabled

- Reduced opacity
- No interaction

### Cards

- Background: white or neutral surface
- Border: subtle rgba line
- Radius: 14–16px
- Minimal shadow depth

### Forms

- Clean outlined inputs
- Cyan focus ring
- Gold used only for validation highlights
- Error states in red

### Tables

- Compact spacing for dense data
- IBM Plex Sans preferred
- Sticky headers optional
- Clear row separation via subtle borders

### Charts

- Minimal grid lines
- Cyan primary series
- Gold used for comparison highlights
- Neutral axis styling

## Motion

- Subtle transitions only
- Fast interaction response
- No exaggerated animations

## Responsiveness

- Desktop-first but fully adaptive
- Tablet layouts optimized for workflow visibility
- Mobile views simplified, not feature-reduced

## Accessibility

- High contrast compliance
- Clear focus indicators
- Readable font scaling
- Keyboard navigation support

## Do Not

- Do not introduce extra colors outside system
- Do not overuse gold accents
- Do not clutter layouts
- Do not increase padding back to default enterprise spacing
- Do not distort or recreate logo outside asset source
